---
description: 今日のブリーフィング — 予定・関連クライアント・滞留タスク・今日のテーマを統合して提示する
---

「AI 社員の作り方」第 4 章「朝にこう聞ける」に対応する応用コマンドです。`morning-summary` ジョブが Daily Note を埋めるのとは別に、より広い範囲を横断したブリーフィングを画面に出します。

実行手順:

1. 今日の Daily Note (`Daily Notes/YYYY-MM-DD.md`) があれば Read。
2. サブエージェント `task-curator` をモード B (滞留検出) で呼び出し、結果を受け取る。
3. サブエージェント `client-tracker` をモード A (横断レビュー) で呼び出し、結果を受け取る。
4. 直近 1 件の `Daily Notes/*.md` (今日以外) を Read して未処理メモを確認。
5. 以下の構成でユーザーに 1 メッセージで返す:

```
# 今日のブリーフ (YYYY-MM-DD)

## 今日の Daily Note (要点)
- ...

## 期限が近い / 超過したタスク
- ... (task-curator モード B より)

## 要フォローのクライアント
- ... (client-tracker モード A より)

## 今日考えるべきテーマ (提案)
- 2–3 個。Daily Note の同セクションが空ならここを書き込む候補として提示。
```

ファイル書き換えは行わない。「Daily Note の『今日考えるべきテーマ』を上記で埋めますか?」とユーザーに確認のみ行う。

引数 $ARGUMENTS で `write` が渡された場合のみ、Daily Note の該当セクションを Edit で書き込む。
