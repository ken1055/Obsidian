---
description: タスクの取り込み・滞留検出・完了反映を task-curator サブエージェントで実行
---

引数 $ARGUMENTS の解釈:

- 引数なし、または `ingest` → モード A (Inbox / Daily Notes から新規タスクを取り込み)
- `check` → モード B (期限超過・期限近接・担当未定の滞留検出)
- `done <タスク内容>` → モード C (該当タスクを完了に変える)

サブエージェント `task-curator` をモードと引数つきで呼び出してください。

モード A 実行後は、追加された件数を報告し、ユーザーに「コミットしますか?」と確認してください。
モード B はレポートのみ。書き換えは無いのでコミットは不要です。
