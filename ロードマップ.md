## Phase 0：プロダクト検証（今〜3ヶ月）

**目的：「売れる」ことを証明する**

### やること
- [ ] 協力仲介業者2〜3社でパイロット撮影（3〜5物件）
- [ ] SUUMOへのURL掲載が規約上OKか確認（★最重要）
- [ ] KPI計測の仕組みを最初から入れる（後述）
- [ ] 料金プランと契約書を作成する

### SUUMO URL問題への対策
SUUMOがコメント欄への外部URLを禁止・削除するリスクがある。
対策として以下のバックアップを用意する：

| 配布方法 | 難易度 | 備考 |
|---|---|---|
| SUUMOコメント欄にURL記載 | 低 | 規約確認が必要 |
| 物件写真にQRコードを合成 | 中 | SUUMOに依存しない |
| 仲介業者の自社サイトに掲載 | 低 | 即実行可能 |
| at home / Homesに掲載 | 低 | SUUMO以外にも展開 |

→ **SUUMO1本に依存しないこと。複数経路を最初から作る。**

### KPI計測の仕組み（今すぐ入れる）
これが将来の営業の武器になる。

```
仕組み：各物件URLにUTMパラメータ or 固有URLを発行
→ Cloudflare Analytics で閲覧数・滞在時間を計測
→ 仲介業者に「掲載前後の問い合わせ数」を月次で報告してもらう
→ 3ヶ月後に「掲載前 vs 掲載後」の比較データを作る
```

このデータが「問い合わせ数が○倍になった」という具体的な実績になる。

### 達成基準
- パイロット物件5件の掲載完了
- SUUMO掲載方法の確認済み
- 1件以上で「問い合わせ数増加」のデータ取得

## Phase 2：ソロ本格稼働（6〜18ヶ月後）

**目的：ソロの上限（15〜20件/月）まで引き上げる**

### 時間の使い方（月単位）
```
撮影・納品：15件 × 4時間 = 60時間
営業・MTG：20時間
開発・改善：20時間
管理・経理：10時間
──────────────
合計：110時間/月（1日3.6時間平均）
```
→ 学生 or 本業と並行しながらでも成立する水準

### 法人化のタイミング
月売上が**100万円を超えたタイミング**で法人化する。

| 個人事業主 | 法人 |
|---|---|
| 最高税率55% | 実効税率25〜35% |
| 社会的信用が低い | 大手との契約がしやすくなる |
| 設立コスト0 | 設立費用約25万円 |

→ 月100万を超えると法人化で年100万以上の節税効果が出る

|---|---|

## Phase 3：スタッフ雇用・スケール（18〜36ヶ月後）

**目的：撮影スタッフを雇い、月30〜45件体制にする**

### 1人目の雇い方
**タイミング**：月15件が3ヶ月連続で安定したら雇用する

**募集方法**：
- フリーランス（最初はクラウドワークスで業務委託）
- 1物件あたり1〜2万円の出来高制から始める
- マニュアルを作り、品質基準を渡す

**マニュアル化が必須なもの**：
- 撮影チェックリスト（Phase 0で作ったもの）
- NG品質の判断基準と対応フロー
- 納品の手順

## Phase 4：Exit or 継続成長（36〜60ヶ月後）

**目的：資産1億に向けた最終手段を決断する**

### 選択肢A：継続成長でFIRE
- 50社以上・月45件体制を維持
- 年間純利益3,000〜4,000万を積み上げる
- 27〜28歳で1億到達

### 選択肢B：SUUMO・大手との提携 or 売却
- 50社以上の導入実績 + KPIデータを持参してSUUMOへ交渉
- 「公式パートナー」契約 → 一気に全国展開
- または会社ごと売却（ARR × 3〜5倍のバリュエーション）
  - 月400万収益 → 年間ARR4,800万 → 売却価格1.4億〜2.4億

## リスクと対応

| リスク | 発生確率 | 対応 |
|---|---|---|
| SUUMO URL禁止 | 中 | QRコード埋め込み・自社サイト経由に切り替え |
| 仲介業者が費用対効果を感じない | 中 | KPIデータで先に証明してから有料化 |
| 大手（Matterport等）が日本参入 | 低〜中 | 先行者優位・価格と関係性で対抗 |
| 撮り直しで時間ロス | 高 | 現地確認プロセスの徹底（チェックリスト） |
| スタッフの品質ばらつき | 中 | マニュアル化 + 現地確認を義務化 |
